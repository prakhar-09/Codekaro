Why ++i operator is better than i++ operator!

INTRODUCTION

If a person had ever written a for-loop before, then he would have definitely used i++ operator to increment the loop variable. However, have you ever thought, that why we use it like that only?
It's quite clear that the end result of i++ would be one higher than the orignal, but it would be same in the case of ++i and i = i+1 also.
In this blog, I would discuss about two methods of adding 1, i.e i++ or post-increment and ++i or pre-increment and would explain why ++i is better than i++ in most cases.

Post Increment (i++)

It is the most common method used to increment values in a for loop. The post increment operators for variable i looks roughly like this when used in a pseudocode-
int k =i ;
i = i + 1;
return k;
 Since the post increment operator has to return the original value of i and not the incremented value i+1, therefore it has to store the older version of i.
This means that it is storing the older versions of i needlessly and is only increasing the memory of the pseudocode, since in most cases we doesn't generally use the old version of i, and it is simply discarded.

Pre Increment (++i)

This method is much less common than the post increment method and is mostly used by old programmers in languages such as C & C++.
The pseudocode, for ++i or Pre-Increment method for variable i looks roughly as follows-
i = i + 1;
return i;
 Particularly, here we do not have to save the older version of i. We can clearly add to it and then return. This matches better with the typical use case in a for-loop, since we rarely need the old value of i in later stages of the code.

Admonition

After discussing the differences between i++ and ++i, one might easily notice that, since the cached value of i is never used in i++ method, the compiler will just optimize that line away, making both the operators equivalent. This is most likely true for primitive data types like float and integer. However, for more complex types of code, where the + operator is overloaded, the compiler may not be able to safely optimize the caching operation.
So, we can finally conclude and say that ++i or Pre-Increment operator is better than, or at most equal to i++ or Post-Increment operator in most cases. It not only reduces the memory required and makes the execution of code faster but also plays a major role in reducing carbon-footprints.
